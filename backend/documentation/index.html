KejaPrime API Documentation

Welcome to the KejaPrime API documentation. Below are the available endpoints.

1. Users

Register a new user

POST /users/register
Body: { "email": "john@gmail.com", "username": "testuser", "password": "secret", "contact": 2766628 }

Login

POST /users/login
    Body: { "email": "john@gmail.com", "password": "secret" }

Get user profile (Authenticated)

GET /users/me
Headers: {
    "Authorization": "Bearer token" }

Post a profile pic

POST /users/avatar
Headers: {
    "Authorization
    ": "Bearer token" }

2. Properties

Get all properties

GET /properties/

Add a new property (Authenticated)

POST /properties/
Body: { "name": "Apartment", "price": 500, "location": "Nairobi", "description": "Nice place to live"}
Headers: { "Authorization": "Bearer token" }

Get a single property

GET /properties/{property_id}

Upload property image

POST /properties/upload
Body: { "property_id": 1, "file": "image.jpg" }
Headers: {
    "Authorization": "Bearer token" }

3. Bookmarks

Get user bookmarks

GET /bookmarks/
Headers: {
    "Authorization": "Bearer token" }

Add a bookmark

POST /bookmarks/
Body: { "property_id": 1 }
Headers: { "Authorization": "Bearer token" }

4. Comments

Get comments on a property

GET /comments/{property_id}
Headers: {
    "Authorization": "Bearer token" }

Add a comment

POST /comments/
Body: { "property_id": 1, "content": "Nice place!" }
Headers: { "Authorization": "Bearer token" }

5. Admin

Get all users (Admin only)

GET /admin/users
Headers: { "Authorization": "Bearer admin_token" }

For authentication, use a Bearer token in the Authorization header.
